## 1. 测试准备

### 1.1 环境要求

- **微信开发者工具**: 最新稳定版
- **微信小程序基础库**: ≥ 2.10.0
- **操作系统**: Windows / macOS
- **测试设备**: iOS 真机 + Android 真机（推荐）

### 1.2 测试前准备步骤

#### Step 1: 确认分支

```bash
git branch
```

**预期结果**: 当前分支应为 `* feature/任务优先级系统`

#### Step 2: 清理测试环境

在小程序开发工具中：

1. 点击菜单栏 **工具** → **清除缓存** → **清除全部**
2. 点击 **编译** 按钮，重新编译项目
3. 确认控制台无错误信息

#### Step 3: 备份现有数据（可选）

如果已有生产数据，建议先备份：


- 准备一个微信测试账号
- 确保账号未登录过新版本（避免数据污染）

## 2. 数据迁移测试

**目的**: 验证旧格式数据能够自动升级为新格式，添加 `priority` 字段

### 2.1 创建旧格式测试数据

#### Step 1: 打开调试控制台

在小程序开发工具中，点击底部的 **调试器** 标签页

#### Step 2: 插入旧格式数据

在控制台中执行以下代码：

```javascript
// 插入旧格式数据（无 priority 字段）
wx.setStorageSync('todos:v1', [
  {
    id: 'test-old-1',
    title: '旧任务1 - 未完成',
    completed: false,
    createdAt: Date.now() - 86400000, // 1天前
    updatedAt: Date.now() - 86400000
  },
  {
    id: 'test-old-2',
    title: '旧任务2 - 已完成',
    completed: true,
    createdAt: Date.now() - 172800000, // 2天前
    updatedAt: Date.now() - 172800000
  },
  {
    id: 'test-old-3',
    title: '旧任务3 - 待处理',
    completed: false,
    createdAt: Date.now(),
    updatedAt: Date.now()
  }
])

console.log('✅ 旧格式数据插入成功')
```

**预期结果**: 控制台输出 `✅ 旧格式数据插入成功`

#### Step 3: 验证数据格式

```javascript
const todos = wx.getStorageSync('todos:v1')
console.log('任务数量:', todos.length)
console.log('第一个任务:', todos[0])
console.log('是否有 priority 字段:', todos[0].priority !== undefined)
```

**预期结果**:
- 任务数量: 3
- 第一个任务包含 id、title、completed、createdAt、updatedAt 字段
- 是否有 priority 字段: false

### 2.2 执行数据迁移

#### Step 1: 重启小程序

在开发工具中点击 **编译** 按钮，重新编译项目

#### Step 2: 等待小程序启动

观察启动日志，确认无错误信息

## 3. 完整功能测试

### 3.1 测试1: 优先级切换功能

**目的**: 验证左滑后点击"优先级"按钮能够循环切换优先级

#### 测试步骤

1. **创建新任务**
   - 点击"添加"按钮
   - 输入任务标题："测试优先级切换"
   - 点击"保存"

2. **验证默认优先级**
   - 查看任务卡片右上角的优先级图标
   - 应显示 🟡⭐（中优先级）

3. **第一次切换**
   - 左滑任务卡片
   - 点击蓝色"优先级"按钮
   - 观察提示消息和图标变化

4. **第二次切换**
   - 再次左滑任务卡片
   - 点击"优先级"按钮
   - 观察提示消息和图标变化

5. **第三次切换**
   - 再次左滑任务卡片
   - 点击"优先级"按钮
   - 观察提示消息和图标变化

6. **第四次切换（循环验证）**
   - 再次左滑任务卡片
   - 点击"优先级"按钮
   - 验证是否回到初始状态

#### 预期结果

| 操作次数 | 优先级 | 图标 | 提示消息 |
|---------|--------|------|---------|
| 初始状态 | 中 | 🟡⭐ | - |
| 第1次切换 | 高 | 🔴⭐⭐ | 优先级：高 |
| 第2次切换 | 低 | 🟢 | 优先级：低 |
| 第3次切换 | 中 | 🟡⭐ | 优先级：中 |
| 第4次切换 | 高 | 🔴⭐⭐ | 优先级：高 |

#### 验证命令

```javascript
const todos = wx.getStorageSync('todos:v1')
const task = todos.find(t => t.title === '测试优先级切换')
console.log('当前优先级:', task.priority)
```

### 3.2 测试2: 编辑功能

**目的**: 验证左滑后点击"编辑"按钮能够正常跳转和保存

#### 测试步骤

1. **准备测试任务**
   - 创建任务："测试编辑功能"
   - 设置为高优先级

2. **触发编辑**
   - 左滑任务卡片
   - 点击橙色"编辑"按钮

3. **验证跳转**
   - 检查是否跳转到编辑页面
   - 检查标题输入框是否显示原标题

4. **修改内容**
   - 将标题修改为："测试编辑功能（已修改）"
   - 点击"保存"按钮

5. **验证保存**
   - 检查是否返回首页
   - 检查任务标题是否更新
   - 检查优先级是否保持不变

#### 预期结果

- 编辑页面正常显示
- 标题输入框显示原标题
- 保存后返回首页
- 任务标题更新为新值
- 优先级保持为高优先级（🔴⭐⭐）

#### 验证命令

```javascript
const todos = wx.getStorageSync('todos:v1')
const task = todos.find(t => t.title === '测试编辑功能（已修改）')
console.log('任务存在:', !!task)
console.log('优先级保持:', task.priority === 'high')
console.log('updatedAt 已更新:', task.updatedAt > task.createdAt)
```

### 3.3 测试3: 删除功能

**目的**: 验证左滑后点击"删除"按钮能够正常删除任务

#### 测试步骤

1. **准备测试任务**
   - 创建任务："测试删除功能"
   - 设置为低优先级

2. **触发删除**
   - 左滑任务卡片
   - 点击红色"删除"按钮

3. **确认删除**
   - 在弹出的确认对话框中点击"确定"

4. **验证删除**
   - 检查任务是否从列表中消失
   - 检查存储中是否还存在该任务

#### 预期结果

- 删除前弹出确认对话框
- 点击"确定"后任务从列表消失
- 存储中不再存在该任务
- 其他任务不受影响

#### 验证命令

```javascript
const todos = wx.getStorageSync('todos:v1')
const task = todos.find(t => t.title === '测试删除功能')
console.log('任务已删除:', !task)
```

#### 取消删除测试

1. 重新创建任务："测试取消删除"
2. 左滑，点击"删除"
3. 在确认对话框中点击"取消"

**预期结果**: 任务保留在列表中

### 3.4 测试4: 统计功能

**目的**: 验证"今日关注"卡片正确显示高优先级任务数量

#### 测试步骤

1. **清空现有任务**
   - 删除所有任务

2. **创建测试任务**
   - 创建任务："高优先级任务1"，设置为高优先级
   - 创建任务："高优先级任务2"，设置为高优先级
   - 创建任务："中优先级任务1"，设置为中优先级
   - 创建任务："低优先级任务1"，设置为低优先级
   - 创建任务："已完成的高优先级任务"，设置为高优先级，然后标记为完成

3. **查看统计卡片**
   - 查看"今日关注"卡片
   - 检查显示的文字

**今日关注卡片应显示**:
```
今日关注
您有 4 个待办任务（2 个高优先级）
```

**说明**:
- 待办任务数 = 4（排除已完成的 1 个）
- 高优先级数 = 2（只计算未完成的高优先级任务）

#### 验证命令

```javascript
const todos = wx.getStorageSync('todos:v1')
const pendingCount = todos.filter(t => !t.completed).length

console.log('高优先级任务数:', highPriorityCount)
```

**测试3: 空列表**
- 删除所有任务
- 验证"今日关注"卡片不显示

### 5.2 常见问题排查

#### 问题1: 优先级不显示

**可能原因**:
1. 数据未迁移成功
2. 组件未正确引入
3. 样式未加载

**排查步骤**:
```javascript
// 检查数据
const todos = wx.getStorageSync('todos:v1')
console.log(todos[0].priority)

// 检查组件
console.log(this.selectComponent('.swipe-card'))

// 检查样式
// 查看调试器中的 Elements 标签
```

#### 问题2: 滑动无响应

**可能原因**:
1. movable-area 配置错误
2. 事件绑定失败
3. CSS 层级问题

**排查步骤**:
```javascript
// 检查组件数据
console.log(this.data.translateX)

// 检查事件
onMove(e: any) {
  console.log('滑动事件:', e.detail.x)
  // ... 现有逻辑
}
```

#### 问题3: 统计数字不更新

**可能原因**:
1. reloadTodos 未调用
2. 过滤条件错误
3. setData 未执行

**排查步骤**:
```javascript
// 检查统计逻辑
reloadTodos() {
  const todos = repository.list()
  console.log('原始数据:', todos)

  const highPriorityCount = todos.filter(
    (t) => t && !t.completed && t.priority === 'high'
  ).length
  console.log('高优先级数量:', highPriorityCount)

  this.setData({
    todos,
    highPriorityCount,
  })
  console.log('setData 完成')
}
```

### B. 性能测试脚本

```javascript
// ===== 性能测试 =====

console.log('===== 开始性能测试 =====')

// 测试大量数据迁移

const largeData = Array.from({ length: 1000 }, (_, i) => ({

const migrateTime = Date.now() - startTime

```
